Background to the app

I recently had to do text analysis for work. To create a synergy between work and education, I incorporated this in the current app as well.

Furthermore, I liked to mess around with some moderatly advanced shiny options like isolation and a reactive user interface. These two things led to the development of the current app.

Functionality

In short the functionality of the app can be divided into three chunks:

  1. Retrieving possible twitter locations and their current trending tags
  2. Retrieving tweets based on the the location and trending tag selected by the user
  3. Processing the text of the tweets through the tm library
  4. Turning the text into a wordcloud

Functionality continued

One advanced functionality in the current app is that the user interface is updated by the location as selected by the user.

Because the location takes processing, we want to isolate the fetchTweets() function otherwise it already runs before the user has selected as tag.

To run the word cloud, the user has to press the go button. This button will also update the fetched tweets if the user provides new input.

Future developments

The following future developments are possible:

  1. I could update the locations specified by the user. For instance, I could use geocoding such that the location is more like an area
  2. I could create some tag networks as well. For instance, it may be interesting to see which words are strongly interrelated. See the next page for such a network.
